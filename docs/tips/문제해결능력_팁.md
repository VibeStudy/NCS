# 문제해결능력 풀이 팁

## 명제 (True/False) 문제

**풀이 순서**
1. 문제에 주어진 명제를 `A → B` 형태로 정리
2. **대우(`~B → ~A`)를 바로 옆에 함께 표시**
3. 정리된 명제들을 연결해서 결론 도출

---

**예시**

> `A를 좋아하는 사람은 B를 좋아한다.` → A → B = → ~B → ~A

---

## 순서·위치 추론 문제

**풀이 시작점: 가장 많이 언급된 사람을 찾아라**

조건들에서 언급 횟수가 많은 사람일수록 제약 조건이 많다
→ 이 사람의 위치를 먼저 확정하면 나머지가 자동으로 좁혀짐

---

**예시**

```
A. D는 첫 번째나 마지막 발표자는 아니었어
B. 나(B)와 A는 연속된 순서로 발표했어
C. 나(C)와 B 사이에는 한 명이 발표했어
D. 나(D)는 E와 연속된 순서로 발표했어
E. B는 C보다 먼저 발표했어
```

| 사람 | 언급된 조건 | 횟수 |
|------|------------|------|
| B | B, C, E | 3회 |
| C | C, E | 2회 |
| D | A, D | 2회 |
| E | D, E | 2회 |
| A | B | 1회 |

→ **B가 3회로 가장 많음** → B의 위치부터 케이스 나열 시작

> 케이스 나열 시 끝자리부터 1씩 증가하며 순서대로 정리 (수리능력 팁 참고)

---

## 반드시 옳은 문제

**풀이 순서**
1. **확정적인 정보를 먼저 찾는다** → 조건에서 위치·순서가 딱 하나로 정해지는 것
2. **케이스를 나눌 수 있는 정보를 찾는다** → 자리 문제에서 2명의 순서가 연속으로 주어지는 경우 등
3. 1, 2를 기반으로 나머지를 조립하며 가능한 케이스를 나열
4. 케이스 전체에서 **항상 성립하는 선지만 정답**, 하나라도 반례가 있으면 제거
