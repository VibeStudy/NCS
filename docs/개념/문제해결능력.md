## 사고력

### 창의적 사고
- 기존의 틀을 벗어나 새롭고 유용한 아이디어를 만들어내는 사고
- 특징: 독창성, 유창성(많은 아이디어), 유연성(다양한 관점), 정교성
- 창의적 사고를 방해하는 요인: 고정관념, 습관, 편견, 관습

#### 발상법 종류

| 구분 | 방법 | 특징 |
|------|------|------|
| 자유연상법 | 브레인스토밍 | 자유롭게 아이디어를 내고 비판 금지 |
| 강제연상법 | 체크리스트법 | 질문 목록으로 강제로 아이디어 유도 |
| 비교발상법 | NM법, 시네틱스 | 다른 것과 비교·유추하여 아이디어 도출 |

##### 자유연상법
- 생각나는 것을 자유롭게 연상하여 아이디어를 얻는 방법
- 대표: **브레인스토밍(Brain Storming)**
  - 4원칙: 비판 금지 / 자유 분방 / 질보다 양 / 결합·편승 환영
- 기타: 브레인라이팅(서면 브레인스토밍)

##### 강제연상법
- 각종 힌트에 강제로 연결 지어 아이디어를 얻는 방법
- 대표: **체크리스트법(Checklist)**
  - 오스본의 체크리스트: 전용, 응용, 변경, 확대, 축소, 대용, 재배열, 역전, 결합 등 9가지 질문
- 기타: 속성열거법, 입출력법

##### 비교발상법
- 주제와 유사한 것을 비교하여 아이디어를 얻는 방법
- 대표: **NM법** — 자연·사회에서 유추하여 발상
- **시네틱스(Synectics)** — 서로 관계없어 보이는 것을 연결해 새로운 아이디어 도출
  - 유추 방식: 직접 유추 / 의인 유추 / 상징 유추 / 환상 유추

### 논리적 사고

- 의미
    - 사고 전개에 있어서 전후관계가 일치하는지 살피고 아이디어를 평가하는 능력 
    - 자신이 만든 계획이나 주장을 주위 사람에게 이해시켜 실현하기 위해 필요한 능력

- 구성 요소
    - 생각하는 습관 
    - 상대 논리의 구조화 
    - 구체적인 생각
    - 타인에 대한 이해 
    - 설득 

- 개발 
    - 피라미드 구조 방법: 하위 사실/현상부터 사고함으로써 상위 주장을 만들어가는 방법
    - So what 방법: 눈앞에 있는 정보로부터 의미를 찾아내어 가치 있는 정보를 끌어내는 방법 

### 비판적 사고

- 의미
    - 어떤 주제나 주장 등에 대해서 적극적으로 분석하고 종합하여 평가하는 능동적인 사고능력 
    - 어떤 논증, 추론, 증거, 가치를 표현한 사례를 타당한 것으로 수용할 것인가 아니면 불합리한 것으로 거절할 것인가에 대한 결정을 내릴 때 요구되는 사고 능력 

- 비판적 사고를 위한 태도 
    - 지적 호기심, 객관성, 개방성, 융통성, 지적 회의성, 지적 정직성, 체계성, 지속성, 결단성, 다른 관점에 대한 존중
    - 비판적인 사고를 개발하기 위해서는 어떤 현상에 대해서 문제의식을 가지고, 고정관념을 버려야함. 

## 문제처리능력 

### 문제 인식 단계

- 의미
    - 문제해결 과정 중 What을 결정하는 단계
    - 해결해야 할 전체 문제를 파악하여 우선순위를 정하고 선정문제에 대한 목표를 명확히 하는 단계 

- 절차

| 단계 | 내용 |
|------|------|
| 환경 분석 | • Business System상 거시적 환경 분석<br>• 문제발생 시 가장 먼저 고려<br>• 3C 분석<br>• SWOT 분석 |
| 주요 과제 도출 | • 환경 분석을 통해 현상을 파악한 후 분석 결과를 검토하여 주요 과제를 도출<br>• 다양한 과제 후보안을 도출하는 일이 선행되어야 함<br>• 과제안 간의 수준이 동일한지, 표현은 구체적인지, 주어진 기간 내에 해결 가능한지 등을 확인 |
| 과제 선정 | • 과제안 중 효과 및 실행 가능성 측면을 평가하여 우선순위를 부여한 후 가장 우선순위가 높은 안을 선정<br>• 우선순위 평가 시에는 과제의 목적, 목표, 자원 현황 등을 종합적으로 고려하여 평가 |

#### 3C 분석

| 구분 | 내용 |
|------|------|
| Customer (고객) | 고객의 니즈, 시장 규모, 구매 행동 분석 |
| Competitor (경쟁자) | 경쟁사의 강점·약점, 시장 점유율, 전략 분석 |
| Company (자사) | 자사의 강점·약점, 보유 자원, 핵심 역량 분석 |

#### SWOT 분석

| 구분 | 내용 |
|------|------|
| Strength (강점) | 경쟁사 대비 자사의 우위 요소 (내부 긍정) |
| Weakness (약점) | 자사의 부족하거나 불리한 요소 (내부 부정) |
| Opportunity (기회) | 외부 환경에서 활용 가능한 유리한 요소 (외부 긍정) |
| Threat (위협) | 외부 환경에서 자사에 불리한 요소 (외부 부정) |

### 문제 도출 단계

- 의미
    - 선정된 문제를 분석하여 해결할 것이 무엇인지 명확히 하는 단계
    - 현상에 대해 문제를 분해하여 인과관계 및 구조를 파악하는 단계

- 절차
ㅣ
| 단계 | 내용 |
|------|------|
| 문제 구조 파악 | • 전체 문제를 작고 다룰 수 있는 이슈들로 세분화하는 과정<br>• 문제의 내용 및 미치고 있는 영향 등을 파악하여 문제 구조를 도출<br>• **Logic Tree**: 문제의 원인을 깊이 파고들거나 해결책을 구체화할 때 제한된 시간 속에 넓이와 깊이를 추구하는데 도움이 되는 기술로, 주요 과제를 나무 모양으로 분해·정리하는 기술<br>• Logic Tree 작성 시 유의사항: 전체 과제를 명확히 / 분해해 가는 가지의 수준을 맞춰야 함 / 원인이 중복되거나 누락되지 않고 각각의 합이 전체를 포함해야 함 |
| 핵심 문제 선정 | • 문제에 큰 영향력을 미칠 수 있는 이슈를 핵심 이슈로 설정 |

### 원인 분석 단계

- 의미
    - 파악된 핵심 문제에 대한 분석을 통해 근본 원인을 도출하는 단계

- 절차

| 단계 | 내용 |
|------|------|
| 이슈 분석 | • **핵심 이슈 설정**: 현재 수행하고 있는 업무에 가장 크게 영향을 미치는 문제로 설정<br>• **가설 설정**: 이슈에 대해 자신의 직관, 경험, 지식, 정보 등에 의존하여 일시적인 결론을 예측해보는 가설을 설정<br>• **가설 설정의 조건**: 관련 자료·인터뷰 등을 통해 검증 가능해야 함 / 간단명료하게 표현해야 함 / 논리적이며 객관적이어야 함<br>• **Output 이미지 결정**: 가설 검증 계획에 의거하여 분석 결과를 미리 이미지화 |
| 데이터 분석 | • **데이터 수집 계획**: 데이터 수집 범위 결정<br>• **데이터 정리/가공**: 정보를 항목별로 정리<br>• **데이터 해석**: What, Why, How 측면에서 의미 해석 |
| 원인 파악 | • 이슈와 데이터 분석을 통해 얻은 결과를 바탕으로 최종 원인을 확인하는 단계<br>• **인과관계의 종류**<br>&nbsp;&nbsp;- **단순한 인과관계**: 원인과 결과를 분명하게 구분할 수 있는 경우<br>&nbsp;&nbsp;- **닭과 계란의 인과관계**: 원인과 결과를 구분하기 어려운 경우<br>&nbsp;&nbsp;- **복잡한 인과관계**: 두 가지 유형이 서로 얽혀있는 경우 |

## 해결안 개발 단계

- 의미
    - 문제로부터 도출된 근본 원인을 효과적으로 해결할 수 있는 최적의 해결방안 수립

- 절차

| 단계 | 내용 |
|------|------|
| 해결안 도출 | • 열거된 근본 원인을 어떠한 시각과 방법으로 제거할 것인지를 명확히 해야 함<br>• 독창적이고 혁신적인 방안을 도출<br>• 전체적인 관점에서 해결 방향과 방법이 같은 것을 그룹핑<br>• 최종 해결안 정리 |
| 해결안 평가 및 최적안 선정 | • 문제, 원인, 방법을 고려해 해결안을 평가하고 가장 효과적인 해결안을 선정<br>• 중요도와 실현 가능성(개발 기간, 개발 능력, 적용 가능성) 등을 고려해 종합적인 평가를 내리고 채택 여부를 결정 |

## 실행 및 평가 단계

- 의미
    - 해결안 개발을 통해 만들어진 실행계획을 실제 상황에 적용
    - 해결안을 사용하여 장애가 되는 문제의 원인을 제거

- 절차

| 단계 | 내용 |
|------|------|
| 실행계획 수립 | • What, Why, When, Where, Who, How의 물음에 대한 답을 가지고 계획<br>• **실행 수립 시 고려사항**<br>&nbsp;&nbsp;- 자원(시간, 예산, 물적, 인적)을 고려<br>&nbsp;&nbsp;- 해결안별 난이도를 고려해 세부 실행 내용을 구체적으로 수립<br>&nbsp;&nbsp;- 실행의 목적과 과정별 진행 내용을 정리 |
| 실행 및 Follow-up | • 가능한 사항부터 실행하며 문제점을 해결해 가면서 해결안의 완성도를 높이고 일정한 수준에 도달하면 전면적으로 전개<br>• **실행 단계에서의 문제점 해결 방안**<br>&nbsp;&nbsp;- Pilot Test를 통해 문제점 발견<br>&nbsp;&nbsp;- 해결안 보완 후 대상 범위를 넓혀서 전면적으로 실시<br>&nbsp;&nbsp;- 실행상 문제점을 해결하기 위해 모니터링 체제 구축<br>• **모니터링 체제 구축 시 유의사항**<br>&nbsp;&nbsp;- 바람직한 상태가 달성되었는지 확인<br>&nbsp;&nbsp;- 문제가 재발하지 않을 것을 확신할 수 있는지 확인<br>&nbsp;&nbsp;- 사전에 목표한 기간 및 비용은 계획대로 지켜졌는지 확인<br>&nbsp;&nbsp;- 또 다른 문제를 발생시키지 않았는지 확인<br>&nbsp;&nbsp;- 해결책이 주는 영향은 무엇인지 확인 |

## 문제풀이유형 

### 명제

- **정의**: 참 또는 거짓을 명확히 판별할 수 있는 문장

| 종류 | 기호 | 의미 | 참이 되는 조건 |
|------|------|------|----------------|
| 부정명제 | `~p` | p가 아니다 | p가 거짓일 때 |
| 합접명제 | `p ∧ q` | p이고 q이다 | p, q 모두 참일 때 |
| 이접명제 | `p ∨ q` | p이거나 q이다 | p, q 중 하나라도 참일 때 |
| 조건명제 | `p → q` | p이면 q이다 | p가 참이고 q가 거짓일 때만 거짓 |

- **조건명제의 변형** (`p → q` 기준)
  - 역: `q → p`
  - 이: `~p → ~q`
  - 대우: `~q → ~p` (원래 명제와 참/거짓이 항상 동일)

- **풀이법**
  - 대우를 활용해 명제의 참/거짓 판단
  - 삼단논법: `p → q`, `q → r` 이면 `p → r`
  - 벤다이어그램으로 포함 관계 시각화
